WebSocket 协议说明 /api/v1/realtime/transcriptions

        
Swagger/OpenAPI 不能直接描述 WebSocket 操作，下面是从服务端 /api/v1/realtime/transcriptions/protocol 获取的协议详情。

      

      

        

          📡 协议详情
          
        

        

          

            
endpoint

            
/api/v1/realtime/transcriptions

            
鉴权：服务端启用 API_TOKEN 时，URL 追加 ?token=your-token

          

          

            
audio_frame

            
binary · 16 kHz · 单声道 · signed int16 little-endian PCM

          

          

            
flow（交互流程）

            
    
              
  • 建立 WebSocket 连接
    • 
              
  • 发送 session.start JSON 配置帧
    • 
              
  • 持续发送 PCM 二进制音频帧
    • 
              
  • 停止录音时发送 audio.end JSON 控制帧
    • 
              
  • 服务端返回 session.started、transcript.delta、transcript.segment、speaker.update 或 error
    • 
            

          

          

            
session.start 示例

            
{
  "type": "session.start",
  "mode": "2pass",
  "audio_fs": 16000,
  "wav_format": "pcm",
  "chunk_size": [0, 10, 5],
  "chunk_interval": 10,
  "encoder_chunk_look_back": 4,
  "decoder_chunk_look_back": 1,
  "itn": true,
  "features": {
    "diarization": true,
    "speaker_match": {"enabled": true, "group_id": "grp_xxx"},
    "emotion": true,
    "events": true,
    "punctuation": true,
    "raw": false
  },
  "options": {
    "language": "auto",
    "hotwords": {"FunASR": 20}
  },
  "fallback": "auto"
}

          

          

            
audio.end 示例

            
{ "type": "audio.end" }

          

          

            
服务端事件

            
    
              
  • session.started — 配置已生效，返回模式、音频格式和能力开关
    • 
              
  • transcript.delta — online/2pass 实时中间文本，一边说一边出字
    • 
              
  • transcript.segment — offline/2pass 最终片段，VAD 断句后先返回文本和临时说话人
    • 
              
  • speaker.update — 声纹匹配完成后异步修正已有文本的说话人标签
    • 
              
  • error — 配置或推理错误
    • 
            

          

          

            
三种模式对比

            
    
              
  • online：只返回 transcript.delta，低延迟；不返回说话人/声纹/情感/事件/完整标点
    • 
              
  • offline：VAD 断句后返回 transcript.segment；可返回增强字段，但无实时中间文本
    • 
              
  • 2pass（推荐）：先返回 delta，断句后返回 segment；声纹命名通过 speaker.update 异步修正
    • 
            

          

          

            
speaker_consistency

            
说话人编号一致性仅在单个 WebSocket 会话内维护。开启 diarization 后服务端用会话级 SpeakerTracker 尽量保持 speaker_0 / speaker_1 跨多句一致；开启 speaker_match 并提供 group_id 后，最终段先显示临时说话人，声纹匹配完成后再用 speaker.update 返回姓名、分数和诊断摘要。
